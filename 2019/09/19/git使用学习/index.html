git使用学习

发表于 2019-09-19 | 分类于 Git

Git作为分布式系统管理工具，可以方便我们对代码进行修改，维护，保存。在个人以及团队的代码管理中都特别受欢迎，掌握Git工具的使用是非常有必要的一件事。无论是对我们的学习还是以后的工作。

通过网上的教程，将一些命令以及作用记录保存下来，希望能有所帮助，更希望以博客的形式督促我学习。

git init

git init

将当前目录初始化为git仓库，生成.git隐藏目录，负责跟踪管理版本，所有有关此项目的快照数据都存放在此处。不要轻易修改，否则可能会破坏git仓库。

使用我们指定目录作为Git仓库。

git init dir_name

git clone

拷贝一个git仓库到本地当前目录下，然后就可以在本地查看并修改项目。

git clone url

url为想要复制的项目。

也可将项目拷贝到指定目录下：

git clone repo directory

repo为Git仓库，directory为本地目录

git clone 时，可以所用不同的协议，包括 ssh, git, https 等，其中最常用的是 ssh，因为速度较快，还可以配置公钥免输入密码。各种写法如下：

git add

将文件添加到缓存。这里需要说明以下，.git目录是版本库(Repository),其中有缓存区和初始化时自动创建的第一个分支master。

而使用git将文件提交到版本库需要两个步骤：

第一步：将文件添加到缓存区，使用命令git add

第二步：将缓存区的所有文件全部提交到当前分支上，使用命令git commit

git add filename

要使用git命令对文件进行操作，就需要将此文件添加到缓存区。

filename为要添加的文件名。

也可使用参数.添加当前目录下的全部文件：

git add .

也可将指定后缀名的文件添加进入缓存区

git add *+后缀名

git status

显示你上次提交更新后的更改或者写入缓存的改动

git status -s

其中-s为参数，是使获得的结构简短输出，如果不加该参数会详细输出内容。

git diff

查看执行git status 的详细信息，git status显示上次进行的改动，git diff将这些改动的具体内容一行一行的显示出来。

git diff filename显示对filename文件的修改

git diff 命令显示已写入缓存与已修改但尚未写入缓存的改动的区别。

  • 查看尚未缓存的改动：git diff
  • 查看已缓存的改动：git diff –cached
  • 查看已缓存与为缓存的所有改动:git diff HEAD
  • 显示摘要而非整个diff：git diff –stat

git commit

上文已经提到过，此命令是将缓存区的全部文件提交到当前分支下。

git需要记录每一次提交的名字和邮箱地址，如果是第一次使用需要配置用户名和邮箱地址，可按照提示输入。

git commit -m "注释"
可在引号内提供提交注释

git reset HEAD

取消已缓存的内容，仅取消当次缓存区的修改
git reset HEAD filename可指定要取消缓存区修改的文件

git rm

如果只是简单地从工作目录中手工删除文件，运行 git status 时就会在 Changes not staged for commit 的提示。

要从 Git 中移除某个文件，就必须要从已跟踪文件清单中移除，然后提交。可以用以下命令完成此项工作:

git rm <file>

如果删除之前修改过并且已经放到暂存区域的话，则必须要用强制删除选项 -f

git rm -f <file>

如果把文件从暂存区域移除，但仍然希望保留在当前工作目录中，换句话说，仅是从跟踪清单中删除，使用 –cached 选项即可

git rm --cached <file>

可以递归删除，即如果后面跟的是一个目录做为参数，则会递归删除整个目录中的所有子目录和文件：

git rm -r *

进入某个目录中，执行此语句，会删除该目录下的所有文件和子目录。

git mv

移动或重命名一个文件、目录、软连接。

git mv cur_name modify_name

cur_name为要修改的文件名，modify_name为修改后文件名。
