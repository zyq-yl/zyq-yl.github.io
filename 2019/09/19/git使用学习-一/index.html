git使用学习（一）

发表于 2019-09-19 | 更新于 2019-09-22 | 分类于 Git

Git作为目前世界上最先进的分布式版本控制系统（没有之一），不但能自动记录每次文件的改动，还可以让同事协作编辑，这样就不用自己管理一堆类似的文件了，也不需要把文件传来传去。可以方便我们对代码进行修改，维护，保存。在个人以及团队的代码管理中都特别受欢迎，掌握Git工具的使用是非常有必要的一件事。无论是对我们的学习还是以后的工作。

本文主要参考了 https://www.liaoxuefeng.com/wiki/896043488029600/896067074338496 的教程，简化提取了一些基本的要点。

创建版本库

git init

git init

将当前目录初始化为git仓库，生成.git目录，（如果看不到此目录说明此目录是隐藏目录），负责跟踪管理版本，所有有关此项目的快照数据都存放在此处。不要轻易修改，否则可能会破坏git仓库。

使用我们指定目录作为Git仓库。

git init dir_name

将文件添加到版本库

这里需要说明以下，创建版本库后的生成.git目录是版本库(Repository),其中有缓存区和初始化时自动创建的第一个分支master。

而使用git将文件提交到版本库需要两个步骤：

第一步：将文件添加到缓存区，使用命令git add

第二步：将缓存区的所有文件全部提交到当前分支上，使用命令git commit

git add

将文件添加到缓存。
git add filename

要使用git命令对文件进行操作，就需要将此文件添加到缓存区。

filename为要添加的文件名。

也可使用参数.添加当前目录下的全部文件：

git add .

也可将指定后缀名的文件添加进入缓存区

git add *+后缀名

git commit

上文已经提到过，此命令是将缓存区的全部文件提交到当前分支下。

git需要记录每一次提交的名字和邮箱地址，如果是第一次使用需要配置用户名和邮箱地址，可按照提示输入。

git commit -m "注释"
可在引号内提供提交注释

每一次修改文件后，若想要提交修改，就要先将文件添加到缓存区，再提交到版本库。

显示工作区的状态和修改内容

git status

显示你上次提交更新后的更改或者写入缓存的改动

git status -s

其中-s为参数，是使获得的结构简短输出，如果不加该参数会详细输出内容。

git diff

查看执行git status 的详细信息，git status显示上次进行的改动，git diff将这些改动的具体内容一行一行的显示出来。

git diff filename显示对filename文件的修改

git diff HEAD --filename查看工作区和版本库中的filename文件的区别。

git diff 命令显示已写入缓存与已修改但尚未写入缓存的改动的区别。

  • 查看尚未缓存的改动：git diff
  • 查看已缓存的改动：git diff –cached
  • 查看已缓存与为缓存的所有改动:git diff HEAD
  • 显示摘要而非整个diff：git diff –stat

git log

git log显示从最近三次的提交日志。

如果觉得输出信息太多，可以加上参数–pretty=online显示简略信息

git log --pretty=oneline

版本回退

git reset HEAD

取消已缓存的内容，仅取消当次暂存区的修改

HEAD是指向当前版本的指针。git回退版本就是将HEAD指针移到回退的版本。

git reset HEAD filename可指定要取消缓存区修改的文件

git reset --hard HEAD^将当前版本回退到上一个版本，如果要回退到上两个版本则加两个^^,如果要回退的版本太多，如回退100个版本就写成HEAD~100。

但是如果回退之后感觉后悔了，向找回最新的版本怎么办呢？

git为我们的每一次提交都生成了版本号(commit id)，通过使用版本号我们就可以重新返回到原来的版本。使用命令：

git reset --hard commit_id，其中commit_id为版本号，没必要写全，但也不能只写前一两位，不然git可能会找到多个版本号，不能确定了。

命令git reflog记录了每一次命令，通过这个命令可以帮助我们找到版本号。

撤销修改

git checkout -- filename将filename文件在工作区的修改全部撤销，这里的文件filename至少已经向版本库提交过一次。

这里有两种情况：

一种是文件自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；

一种是文件已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。

总之，就是让这个文件回到最近一次git commit或git add时的状态。

但是如果你修改了文件，又将它git add进了暂存区,那么在提交之前，你可以使用下面的命令将暂存区的修改回退到工作区

git reset HEAD -- filename

然后在使用git cheackout -- filename将工作区的修改撤销

总之

  • 当你在工作区修改了某个文件的内容，向丢弃工作区的修改时，使用命令git checkout -- filename
  • 当你在工作区修改了某个文件的内容，并且又将其添加到暂存区后，若想撤销修改分为两步，第一步，使用命令git reset HEAD filename返回上一个场景，再按照上一个场景的方法操作。
  • 如果提交了不合数的版本，可按照版本回退的操作撤销修改。

删除文件

git rm

如果只是简单地从工作目录中手工删除文件，运行 git status 时就会在 Changes not staged for commit 的提示。

要从 Git 中移除某个文件，就必须要从已跟踪文件清单中移除，然后提交。可以用以下命令完成此项工作:

git rm <file>

如果删除之前修改过并且已经放到暂存区域的话，则必须要用强制删除选项 -f

git rm -f <file>

如果把文件从暂存区域移除，但仍然希望保留在当前工作目录中，换句话说，仅是从跟踪清单中删除，使用 –cached 选项即可

git rm --cached <file>

可以递归删除，即如果后面跟的是一个目录做为参数，则会递归删除整个目录中的所有子目录和文件：

git rm -r *

进入某个目录中，执行此语句，会删除该目录下的所有文件和子目录。

使用命令git rm后在使用git commit提交一次，那么文件就从版本库中删除了。

如果你手动删除了文件，但是还没有提交，此时你想恢复文件，使用命令git checkou -- filename可以把文件恢复到最新版本，因为git checkout其实是用版本库的文件替换工作区的文件。

添加远程仓库

github是为git提供仓库托管服务的网站，在github创建了一个仓库后，可以使之与本地仓库同步，这样，github上的仓库既可以作为备份，又可以让其他人通过该仓库来协作。

由于本地仓库和github仓库的传输通过ssh加密，所以首先，需要在自己的github账户上添加SSH KEY密匙：

  • 创建SSH密匙，先查看用户主目录下有无.ssh目录（Windows下可在C:\Users\ad.ssh下找到，Linux下可直接cd ~/.ssh），如果有再看看有无id_rsa和id_rsa.pub这两个文件，如果有直接进入下一步，若没有则使用命令$ ssh-keygen -t rsa -C "youremail@example.com"输入自己的邮箱地址，一路回车即可。
    如果一切顺利的话，可以在用户主目录里找到.ssh目录，里面有id_rsa和id_rsa.pub两个文件，这两个就是SSH Key的秘钥对，id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人。
  • 登陆GitHub，打开“Account settings”，“SSH Keys”页面。然后，点“Add SSH Key”，填上任意Title，在Key文本框里粘贴id_rsa.pub文件的内容。如果添加多台电脑的SSH密匙就可以在多台电脑上向github上推送。

接下来便可以添加远程仓库了。

首先新建一个仓库，在本地git仓库下运行命令：

git remote add origin git@github.com:username/reponame.git

输入自己的github账户名和仓库名，就可以将本地仓库与之关联。

添加后，远程库的名字就是origin，这是Git默认的叫法，也可以改成别的，但是origin这个名字一看就知道是远程库。

下一步，将本地仓库的所有内容推送到远程库上：

git push -u origin master

把本地库的内容推送到远程，用git push命令，实际上是把当前分支master推送到远程。

由于远程库是空的，我们第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

刷新github仓库页面，就可以看到已经与本地仓库一样了。

此后，只要在本地做了修改，便可以通过命令git push origin master将本地master分支的最新修改推送至github。

从远程库克隆

git clone

拷贝一个git仓库到本地当前目录下，然后就可以在本地查看并修改项目。

git clone url

url为想要复制的项目。

也可将项目拷贝到指定目录下：

git clone repo directory

repo为Git仓库，directory为本地目录

如果有多个人协作开发，那么每个人各自从远程克隆一份就可以了。

git clone 时，可以用不同的协议，包括 ssh, git, https 等，其中最常用的是 ssh，因为速度较快，还可以配置公钥免输入密码。各种写法如下：

请他喝杯茶
