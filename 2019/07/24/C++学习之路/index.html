前言

作为一名计算机的学生，应当具有良好的总结和写作能力，希望通过写博客的方式，能帮助我提高这些能力，并帮助我理清和加深对所学知识的掌握。作为一位小白，希望我浅显和并不专业的知识与表达能够对像我一样的初学者有所帮助。

接下来便是我的c++学习之路，欢迎斧正共同学习。

初识C++

相信大家学过c的都明白，c语言的难点与灵魂在于指针。其实指针也就是一个存储地址的变量，但当我们去使用，去理解的时候却发现几乎我们所学的全部内容与指针都相关，程序的好坏和效率与指针的使用也十分相关。指针的灵活为我们提供了各种各样的解决方案，但灵活也使得可能造成的bug也繁多且难以排查，其他的高级语言如JAVA和python则舍弃了指针 （其实是将指针的使用封装了起来，给用户仅提供了API来使用），使得开发的效率大大提高。

而c++最初被称为面向对象的c语言，从名称上我们便可以知道，c++继承了c的一切，并在此基础上添加了面向对象的内容。在主流的高级语言中，c++被公认为是最难的语言，一方面是因为其保留的c语言的指针和添加的引用等内容，另一方面就是因为其内存管理的方面。不同于JAVA和python拥有垃圾回收机制的语言，c++的内存管理需要程序员自己管理，这方面的内存泄漏和野指针问题也是令c++程序员特别头痛的问题。

说到这里似乎都是在讲述c++的缺点，那么为什么还要学习c++？最主要的原因便是 效率 ，我相信没人愿意忍受游戏时时不时地卡顿，也不愿意漫长的等待界面，更不愿意使用时的无响应。特别是对庞大的数据进行处理时，c++是非常推荐的语言工具。

更多的认识与了解希望在接下来的学习中逐渐深入。

下次便正式进入c++的学习之中。
